# 通知管理功能说明

## 功能概述

本次更新为小智笔记系统添加了完整的**后台通知管理功能**，支持管理员向个人或全体用户发送系统通知。

## 主要特性

### 1. 通知类型
- **系统公告** (type=4): 面向所有用户的重要公告
- **管理员通知** (type=5): 管理员发送给特定用户或全体的通知
- **文章下架** (type=1): 系统自动触发的文章下架通知
- **评论审核** (type=2): 评论审核结果通知
- **新评论** (type=3): 文章收到新评论的通知

### 2. 发送方式
- **全员通知**: 向所有活跃用户发送通知
- **指定用户**: 选择一个或多个特定用户发送通知

### 3. 管理功能
- 查看所有通知记录
- 按用户、类型、来源筛选
- 查看通知详情
- 删除通知
- 统计数据展示

## 技术实现

### 后端改动

#### 1. 数据库模型更新 (`backend/app/models/notification.py`)
```python
class Notification(Base):
    user_id = Column(BigInteger, nullable=True)  # 可空，表示全员通知
    type = Column(SmallInteger)  # 新增类型 4-系统公告, 5-管理员通知
    is_system = Column(Boolean, default=False)  # 是否为系统发送
    created_by = Column(BigInteger, ForeignKey("users.id"))  # 创建者ID
```

#### 2. 新增API接口 (`backend/app/routers/notification.py`)

**管理员发送通知**
```
POST /api/v1/notifications/admin/send
```
请求体:
```json
{
  "title": "系统维护通知",
  "content": "系统将于今晚进行维护...",
  "type": 4,
  "target_user_ids": [1, 2, 3]  // null表示全员
}
```

**获取通知列表**
```
GET /api/v1/notifications/admin/list?page=1&size=10&user_id=1&notification_type=4&is_system=true
```

**删除通知**
```
DELETE /api/v1/notifications/admin/{notification_id}
```

**获取统计数据**
```
GET /api/v1/notifications/admin/stats
```

#### 3. 用户API扩展 (`backend/app/routers/user.py`)
```
GET /api/v1/users/all  # 获取所有用户列表（用于选择接收者）
```

#### 4. 数据库迁移 (`backend/app/database.py`)
自动添加新字段到现有notifications表：
- `is_system`: 标识是否为系统/管理员发送
- `created_by`: 记录创建者ID
- `user_id`: 改为可空以支持全员通知

## 使用指南

### 访问入口
1. 登录管理员账号
2. 进入后台管理系统
3. 点击左侧菜单"通知管理"

### 发送通知步骤
1. 点击右上角"发送通知"卡片或按钮
2. 填写通知标题（必填）
3. 填写通知内容（可选）
4. 选择通知类型：系统公告 或 管理员通知
5. 选择接收对象：
   - **全员通知**: 发送给所有活跃用户
   - **指定用户**: 从下拉框选择一个或多个用户
6. 点击确定发送

### 查看和管理通知
- **筛选**: 可按用户、通知类型、来源类型筛选
- **详情**: 点击"查看"按钮查看完整通知信息
- **删除**: 点击"删除"按钮移除通知记录
- **统计**: 顶部卡片显示关键统计数据

## 数据流程

```
管理员输入 → 前端验证 → API调用 → 后端处理 → 数据库存储
                                        ↓
                                   为每个目标用户创建通知记录
                                        ↓
                                   返回发送成功信息
```

## 注意事项

1. **全员通知**: 当不指定用户时，系统会为所有`status=1`的活跃用户创建独立的通知记录
2. **权限控制**: 只有管理员(role >= 1)才能访问通知管理功能
3. **批量创建**: 发送全员通知时，如果用户量大，可能需要几秒时间处理
4. **已读状态**: 新创建的通知默认为未读状态
5. **关联ID**: 可通过`related_id`字段关联文章或评论等资源

## 数据库表结构

```sql
CREATE TABLE notifications (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT COMMENT '接收用户ID，为空表示全员通知',
    type SMALLINT NOT NULL COMMENT '类型: 1-文章下架, 2-评论审核, 3-新评论, 4-系统公告, 5-管理员通知',
    title VARCHAR(200) NOT NULL,
    content TEXT,
    related_id BIGINT COMMENT '关联ID',
    is_read BOOLEAN DEFAULT FALSE NOT NULL,
    is_system BOOLEAN DEFAULT FALSE NOT NULL COMMENT '是否为系统发送',
    created_by BIGINT COMMENT '创建者ID',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP NOT NULL,
    INDEX idx_user_id (user_id),
    INDEX idx_is_read (is_read),
    INDEX idx_created (created_at),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE SET NULL
);
```
